1. Payment Receipt Automation

Telkomsel Collection Operations — Sulawesi
The Problem

Collection teams received dozens of payment receipts every day via Telegram. Agents had to manually read each photo and type the data into a spreadsheet — one by one. Hours of work, high error rate, and zero consistency across the team.

What I Built

Built an end-to-end n8n workflow. When an agent sends a payment receipt photo via Telegram, Llama 4 Scout (Groq) extracts Channel, Amount, Admin Fee, and Date — validating each field individually. Valid data is appended to Google Sheets with a structured confirmation sent back. If any field fails, the agent is told exactly which one to retake.

Tools Used
n8nTelegram Bot APIGroq (Llama 4 Scout)Google SheetsREST API
Result

Manual data entry dropped to zero. 95% accuracy via Groq (Llama 4 Scout). Processing time cut from hours to under 10 seconds per receipt. Handles 500+ transactions. Per-field error feedback means agents know exactly which photo to retake.

Proof of Work

3. Transfer Dashboard & Reconciliation System

PT Maulagi Indo Solusi — 80+ Branches, 2 Operational Areas (Makassar / Remote)
The Problem

80+ branches across 2 operational areas (SE Sulawesi & Makassar Outer) had no digital system to submit or verify transfer proofs. Multi-bank payments (BRI, BCA, ShopeePay, GoPay, BNI, DANA) were tracked manually — admin had to manually cross-check each transfer receipt, consuming nearly a full working day just for daily reconciliation.

What I Built

Full-stack web application with a hybrid Vercel + AWS Lambda architecture. Branch operators upload transfer proof photos via a web form — Groq AI/OCR auto-extracts bank name, amount, and date. Data is stored in Supabase (PostgreSQL) with deduplication checks. OCR and NONCOD sync use async background pipelines so user requests return instantly while heavy processing runs separately. Admin dashboard provides real-time visibility per branch, per bank, and per period with XLSX export. NONCOD/DFOD monitoring synchronized from MauKirim. AWS Lambda handles operational Telegram notifications and background sync pipelines. Rate limiting enforced per-instance via Upstash Redis. Secured with bcrypt auth and OWASP-aligned HTTP security headers.

Tools Used
Node.jsSupabaseGroq AI / OCRVercelAWS LambdaUpstash RedisbcryptREST APIHTML/CSS/JS
Result

Daily reconciliation time cut from a full working day to under 20 minutes — a 95% reduction. 99% data duplicates eliminated. Rp90M+/month transaction volume monitored in real-time across 80+ branches. Management gets live branch-level accuracy without any manual effort.

Proof of Work
